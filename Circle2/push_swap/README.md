# push_swap

## 프로젝트 개요

push_swap은 제한된 명령어 세트를 사용하여 스택을 정렬하는 알고리즘을 구현하는 프로젝트입니다. 가능한 적은 명령어로 정수들을 정렬해야 합니다.

## 규칙

두 개의 스택(a, b)을 사용합니다:
- 스택 a: 초기에 정렬되지 않은 숫자들이 있음
- 스택 b: 초기에 비어있음

## 사용 가능한 명령어

### Swap 명령어
| 명령어 | 설명 |
|--------|------|
| `sa` | 스택 a의 상위 2개 요소 교환 |
| `sb` | 스택 b의 상위 2개 요소 교환 |
| `ss` | sa와 sb 동시 실행 |

### Push 명령어
| 명령어 | 설명 |
|--------|------|
| `pa` | 스택 b의 top을 스택 a로 이동 |
| `pb` | 스택 a의 top을 스택 b로 이동 |

### Rotate 명령어
| 명령어 | 설명 |
|--------|------|
| `ra` | 스택 a를 위로 회전 (첫 번째가 마지막으로) |
| `rb` | 스택 b를 위로 회전 |
| `rr` | ra와 rb 동시 실행 |

### Reverse Rotate 명령어
| 명령어 | 설명 |
|--------|------|
| `rra` | 스택 a를 아래로 회전 (마지막이 첫 번째로) |
| `rrb` | 스택 b를 아래로 회전 |
| `rrr` | rra와 rrb 동시 실행 |

## 평가 기준

정렬에 사용된 명령어 수에 따라 점수가 결정됩니다:

### 100개 숫자
| 명령어 수 | 점수 |
|-----------|------|
| < 700 | 5점 |
| < 900 | 4점 |
| < 1100 | 3점 |
| < 1300 | 2점 |
| < 1500 | 1점 |

### 500개 숫자
| 명령어 수 | 점수 |
|-----------|------|
| < 5500 | 5점 |
| < 7000 | 4점 |
| < 8500 | 3점 |
| < 10000 | 2점 |
| < 11500 | 1점 |

## 알고리즘 접근법

### 작은 수 (3-5개)
- 하드코딩된 최적 해법 사용

### 중간 수 (약 100개)
- 청크(chunk) 기반 알고리즘
- 삽입 정렬 변형

### 큰 수 (500개)
- 기수 정렬 (Radix Sort)
- 퀵 정렬 변형

## 빌드

```bash
make
```

## 실행

```bash
./push_swap 2 1 3 6 5 8       # 직접 숫자 입력
./push_swap "2 1 3 6 5 8"     # 문자열로 입력

# 결과 검증 (checker 사용)
ARG="4 67 3 87 23"; ./push_swap $ARG | ./checker $ARG
```

## 디버깅 팁

```bash
# 명령어 수 확인
./push_swap 3 2 1 | wc -l

# 랜덤 숫자 생성
ARG=$(shuf -i 1-100 -n 100 | tr '\n' ' ')
./push_swap $ARG | wc -l
```
